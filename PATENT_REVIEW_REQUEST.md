## 1. 신청 개요

본 신청자는 **본인이 직접 운영하는 식당의 운영 데이터를 자동화하기 위해 1인 개발한 PWA(Progressive Web App)**의 핵심 기술에 대해 출원 검토를 의뢰드립니다.

### 신청자 배경
- 1인 자영업자 (식당 운영자 + 풀스택 개발자)
- 본인 식당에서 매일 사용하면서 직접 개발·검증
- 라이브 동작: https://restraunt-ebon-phi.vercel.app
- 소스: https://github.com/jaewonhyeon9-ctrl/restraunt
- 모두의 창업 사업 신청 완료, 결과 대기 중

### 신청 개요
- **이번 신청 출원 대상**: 메뉴 레시피 기반 다중 재고 자동 차감 및 실시간 원가율 산출 방법 (별첨 발명설명서 참조)
- **함께 검토 의뢰**: 추가 청구 후보 12개에 대한 출원 전략 자문

## 3. 선행특허 10-2741384 회피 분석 (검증 부탁드립니다)

### 선행특허 정보
- 등록번호: 10-2741384 (등록일 2024-12-06)
- 명칭: 식당의 원자재 재고 관리 및 발주 시스템
- 출원인: 김경우, 김경윤
- 청구항 1 구성요소:
  - (a) OCR데이터인식 단말기
  - (b) POS수요자서버 (재고 적산 + 부족 알림)
  - **(c) POS공급자서버** (공급업체 측 별도 서버)
  - **(d) POS관리자서버** (공급업체-식당 매칭·추천·자동발주 중개)

### 신청자 시스템과의 차이 분석

| 구성요소 | 선행특허 | 본 발명 (오토드림) |
|---|---|---|
| (a) OCR | 일반 OCR | **멀티모달 LLM** (Gemini Vision) |
| (b) 식당 측 시스템 | POS 단말 + 별도 서버 | **휴대폰 PWA 단일 단말** |
| (c) 공급업체 서버 | **있음** (마켓플레이스 구조) | **없음** |
| (d) 관리자 중개 서버 | **있음** (매칭·추천·자동발주) | **없음** |
| 데이터 방향 | **입고 적산** (구매→재고+) | **판매 차감** (메뉴판매→재고-) |
| 핵심 기술 | 공급업체 우선순위 수식 | **메뉴 레시피 다중 차감 + 원가 스냅샷** |

### 신청자 검토 의견
**구성요소 완비의 원칙**상 (c), (d) 가 누락되어 있고, 데이터 방향이 정반대(IN vs OUT)이며, 핵심 기술 영역이 다르므로 본 발명은 선행특허 10-2741384를 **침해하지 않는다**고 판단됩니다.

### 검토 의뢰 사항
1. 위 침해 여부 판단이 맞는지 확인 부탁드립니다.
2. 청구항을 작성할 때 선행과 명확히 구분되도록 **"공급업체 매칭이 아닌 단일 매장 내부 도구"**, **"입고가 아닌 판매 시점 차감"** 을 한정 요건으로 명시하는 것이 좋을지요?
