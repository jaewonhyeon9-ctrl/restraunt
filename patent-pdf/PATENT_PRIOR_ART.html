2. 영역별 차별화 포인트 분석

2.1 OCR 매출 자동 입력

  • 종래 솔루션 한계:
    • 캐셔플/페이히어: 매출은 POS 연동에만 의존. POS와 직접 연동되지 않으면 수기 입력
    • 바로가게: OCR 있으나 단순 텍스트 추출, 메뉴/금액 구조화 안 됨
    • 더치트: 가계부 형태로만 작동, 식당 도메인 특화 없음
  • 오토드림 차별점:
    • 멀티모달 LLM이 한국 영수증/POS 화면을 도메인 컨텍스트와 함께 처리 → 인식률 90%+
    • JSON 스키마 강제 출력으로 후속 시스템 연결 안정성
    • 부분 인식 결과를 수정 폼에 자동 채움 → 사용자 보정 시간 단축

2.2 OCR → 재고 자동 차감

  • 종래 솔루션 한계:
    • 매출 시스템과 재고 시스템이 분리됨. 사용자가 매출 입력 후 별도로 재고 차감 입력 필요
    • 일부 ERP는 매출-재고 연동 있으나 POS 직접 통신만 지원, OCR 기반 없음
  • 오토드림 차별점:
    • 영수증 사진 1장 → 매출 등록 + 재고 차감 + 알림이 단일 트랜잭션
    • 메뉴-재고 매핑 학습으로 시간이 갈수록 자동화율 상승

2.3 다단계 폴백 입력

  • 종래 솔루션 한계:
    • POS 직접 연동에 의존 → POS 미사용 식당, 다중 매출 채널(매장+배달앱+오프라인) 식당은 불완전
    • 엑셀 업로드만 제공하는 솔루션은 컬럼 형식이 고정되어 사용자가 매번 양식 맞춰야 함
  • 오토드림 차별점:
    • 단일 화면에 5가지 입력 경로가 동시 노출
    • 컬럼명 자동 감지(한국어/영어/속어 사전) → 양식 강제 없음
    • POS 미사용 식당, 배달앱 매출, 오프라인 매출 모두 처리 가능

2.4 AI 자연어 리포트

  • 종래 솔루션 한계:
    • 그래프/표만 제공. "왜 어제 매출이 떨어졌는지" 같은 진단형 텍스트 없음
    • 식당주는 통계 해석 능력이 제각각 → 데이터가 있어도 활용 못 함
  • 오토드림 차별점:
    • 통계 + 도메인 컨텍스트를 LLM에 주입 → 자연어 진단 + 권장 행동
    • 캐시 전략으로 비용 절감

2.5 PWA 단일 코드베이스

  • 종래 솔루션 한계:
    • 네이티브 앱(iOS/Android 별도) — 개발/유지보수 비용 큼, 출시 심사 필요
    • 데스크톱 SW — 식당 내 단말에 묶임, 외부 이동 불가
  • 오토드림 차별점:
    • 단일 PWA로 iOS/Android/데스크톱 모두 지원
    • 앱스토어 등록 없이 URL로 즉시 사용
    • 서비스 워커로 일부 오프라인 가능

3. 특허 등록 가능성 평가 (자체 추정)

청구 후보 신규성 진보성 등록 가능성
① 다단계 폴백 매출 입력 (4-tier) 중 중 △ 청구항 좁히면 가능
② OCR 매출 자동 입력 (멀티모달 LLM) 상 상 ✅ 매우 유망
③ OCR → 재고 자동 차감 (단일 트랜잭션) 상 상 ✅ 매우 유망
④ 메뉴-재고 매핑 자동 학습 상 중상 ✅ 유망
⑤ 이종 데이터 정규화 엔진 중 중 △ 좁혀서 출원
⑥ PWA 통합 운영 시스템 하 중 △ 다른 청구와 결합 시
⑦ AI 자연어 리포트 자동 생성 중상 상 ✅ 유망
⑧ 재고 발주 시점 예측 중 중상 △ 좁혀서 출원

전략 제안:

  • 묶음 출원 1건 — ② + ③ + ④를 결합한 "OCR 기반 식당 매출·재고 자동화 시스템"으로 하나
  • 별도 출원 1건 — ⑦ "AI 자연어 식당 리포트 생성 방법"
  • 나머지(①·⑤·⑥·⑧)는 첫 출원 명세서 본문에 포함시키되 청구항으로는 요약 형태만

5. 차별화 입증을 위한 데모 시나리오

변리사에게 보여줄 라이브 데모 흐름:

  1. 캐셔플 / 페이히어 화면 비교
    • "이 솔루션은 매출 입력하려면 POS 단말 필요" → 한계 시연
  2. 오토드림 라이브 데모
    • 영수증 사진 1장 → 매출 등록 + 재고 차감 + 알림 → 30초 내 완료 시연
    • 엑셀 파일을 그대로 끌어다 놓기 → 컬럼 자동 감지 시연
    • "이번 달 매출 분석" 클릭 → AI 자연어 리포트 자동 생성 시연
  3. 사용자 환경 다양성 시연
    • POS 없는 푸드트럭주
    • POS + 배달앱 3개 병용 식당
    • 영수증만 있는 노점

6. 결론

오토드림의 핵심 발명은 **"이종 매출 데이터를 멀티모달 AI로 정규화하여 단일 트랜잭션으로 매출+재고를 동시에 처리하는 메커니즘"**이며, 이는 한국 식당 운영 SaaS 시장에서 선행 기술이 거의 없는 영역으로 판단됨.

특히 ②·③·④의 결합은 종래 기술 대비 명확한 진보성과 효과를 가지므로 묶음 출원 1건의 핵심 청구항으로 강력 추천.

⑦ AI 자연어 리포트는 별도 출원으로 분리해 권리 범위를 넓히는 전략이 유리.
